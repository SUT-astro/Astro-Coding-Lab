LAB-1: การจำแนกดาวแคระน้ำตาลด้วย Random Forest

แบบจำลอง Random Forest (RF model)

ต้นไม้การตัดสินใจ (Decision tree)

Decision tree เป็นโมเดลที่สามารถใช้ได้ในการแก้ปัญหาการจำแนกประเภท (classification) และการถดถอย (regression) โดยอิงตามลำดับขั้นตอนของต้นไม้ที่สร้างขึ้นจากข้อมูลที่มีอยู่โดยใช้ฟังชันก์ทางคณิตศาสตร์ Decision tree เป็นอัลกอริทึมที่ใช้ในการทำนายผลลัพธ์ของการตัดสินใจ (ในกรณีของเราคือ การจำแนกประเภทของดาวแคระน้ำตาล) โดยโครงสร้างของ Decision tree จะประกอบไปด้วยโหนด (node) ต่าง ๆเชื่อมหากัน

เราเรียกจุดเริ่มต้นของ Decision tree ว่า "ราก" (root) ซึ่งเป็นเงื่อนไขเริ่มต้นในการแบ่งข้อมูลออกเป็นกลุ่มย่อย โหนดย่อยที่เชื่อมต่อกับราก เรียกว่า "โหนดลูก" (child nodes) ซึ่งเป็นเงื่อนไขการแบ่งข้อมูลเพิ่มเติม โหนดที่อยู่ ณ จุดสิ้นสุดของแต่ละเงื่อนไขคือ "โหนดปลาย" (leaf node) ซึ่งจะให้ผลลัพธ์ของการทำนายออกมา

ตัวอย่างใน Figure 1. แสดง decision tree ในการจำแนกสัตว์ 3 ตัว ประกอบด้วย dog, spider, และ penguin โดยโหนดบนสุดของ decision tree คือราก และมีโหนดของเงื่อนไขเป็นโหนดลูก โหนดตรงปลายคือผลการทำนาย ซึ่งในตัวอย่างนี้คือประเภทของสัตว์

RF regression

Random Forest Regression เป็น Machine learning ที่ใช้กลุ่มก้อน (ensemble) ซึ่งประกอบไปด้วย Decision tree หลายต้นในการทำนายข้อมูล โดยการสร้างกลุ่มก้อนของ Decision tree นั้นมาจากกระบวนการ Bootstrapping และ Attribute sampling ซึ่งสองกระบวนการนี้เป็นส่วนสำคัญที่ทำให้ได้ Decision tree แต่ละต้นที่มีความแตกต่างกัน

Bootstrapping คือการสุ่มสับเซตข้อมูลจากชุดข้อมูลต้นฉบับ โดยใช้การสุ่มแบบใส่คืน (Sampling with Replacement) ซึ่งมีความหมายทางคณิตศาสตร์ว่า การสุ่มแต่ละครั้งเป็นอิสระต่อกัน (independent) หมายความว่าข้อมูลบางตัวอาจถูกสุ่มมาหลายครั้ง แต่บางตัวก็อาจจะไม่ถูกสุ่มมาเลย โดยจำนวนครั้งที่ทำการสุ่มนั้นมีค่าเท่ากับขนาดของข้อมูลต้นฉบับ หมายความว่าจำนวนของข้อมูลที่ได้จากการทำ Bootstrapping จะมีขนาดเท่ากันกับจำนวนข้อมูลต้นฉบับ

ตัวอย่างใน Table 1. แสดงจำนวนข้อมูลที่ได้จากการทำ Bootstrapping จะเห็นได้ว่าในบางครั้งเราได้ข้อมูลบางตัวที่ปรากฏซ้ำ (Bootstrap round 1 ได้ sample 4 ออกมา 2 ตัว) ในขณะที่บางครั้งมีข้อมูลบางตัวที่หายไป (Bootstrap round 2 ได้ sample 2, 4, และ 6 ออกมา 0 ตัว)

ผลของจำนวนของข้อมูลจากการทำ Bootstrapping จำนวน 3 ครั้ง ของข้อมูล 6 ตัว

Attribute sampling คือการสุ่มเลือก feature ที่มีความเหมาะในขั้นตอนการสร้างโหนดลูกที่ระบุเงื่อนไขการแบ่งข้อมูล หมายความว่าในขั้นตอนการสร้างโหนดลูกไม่ได้พิจารณาทุก feature แต่จะสุ่มเลือกแค่บาง feature มาพิจารณาเท่านั้น ก่อนทำการหา feature ที่ดีที่สุด

ตัวอย่างใน Figure 2. แสดง Attribute sampling ของข้อมูลที่มี 5 feature: f1, f2, f3, f4, และ f5 โดย feature ที่สุ่มมาใช้ในการพิจารณาคือสีน้ำเงิน ส่วนสีขาวคือ feature ที่ไม่ได้พิจารณา กรอบสีแดงแสดง feature ที่ดีที่สุดที่ถูกเลือกในการสร้างเงื่อนไข

สับเซตข้อมูลที่ได้จากสองกระบวนการนี้จะถูกนำมาสร้าง Decision trees จากนั้นเราทำกระบวนการเหล่านี้ซ้ำไปเรื่อย ๆ เพื่อสร้าง Decision trees ขึ้นมาหลายต้นเพื่อรวมเป็นกลุ่มก้อน (ensemble) เข้าด้วยกันเป็น Random Forest Model (เรียกกระบวนการนี้ถูกเรียกว่า Ensemble learning) ซึ่งมีความแม่นยำมากกว่า และเสี่ยงต่อการเกิด Overfitting น้อยกว่า เมื่อเปรียบเทียบกับการใช้ Decision tree เพียงต้นเดียวดังที่แสดงใน Figure 3.
