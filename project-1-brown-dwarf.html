ดาวแคระน้ำตาล (Brown Dwarfs)

ดาวแคระน้ำตาลคืออะไร

ดาวแคระน้ำตาลคือดาวที่มีมวลมากกว่าดาวเคราะห์ แต่น้อยกว่าดาวฤกษ์ ในทางดาราศาสตร์ ดาวแคระน้ำตาลถูกจัดว่าเป็นวัตถุมวลต่ำกว่าดาวฤกษ์ (substellar object) ซึ่งถูกคาดการณ์การมีอยู่ทางทฤษฎีครั้งแรกโดย Dr. Shiv S. Kumar ในช่วงปี 1960s ในช่วงแรก เราเรียกดาวแคระน้ำตาลว่า “ดาวแคระดำ” อันเนื่องมาจากขนาดที่เล็กและอุณหภูมิที่ต่ำ ทำให้ยากต่อการค้นพบจากการสังเกตการณ์ จนกระทั้งในปี 1994-1995 มีการค้นพบดาวแคระน้ำตาล 2 ดวงคือ Gliese 229b and Teide 1 จากการสังเกตการณ์ในย่านคลื่นอินฟราเรด

ทำไมดาวแคระน้ำตาลจึงถูกค้นพบได้ยาก?

เนื่องจากมวลของดาวแคระน้ำตาลนั้นมีค่ามากกว่ามวลของดาวเคราะห์ แต่น้อยกว่าดาวฤกษ์ (มีมวลระหว่าง 13 ถึง 80 เท่าของดาวพฤหัสบดี) ซึ่งอยู่ในช่วงที่ทำให้เกิดปฎิกริยาฟิวชันของดิวเทอเรียม (Deuterium fusion) ที่แกนกลางของดาว แต่ไม่มากพอที่จะทำให้เกิดเกิดปฎิกริยาฟิวชันของไฮโดรเจนแบบดาวฤกษ์ (มีมวล > 80 เท่าของดาวพฤหัส) ทำให้บางครั้งนักดาราศาสร์เรียกมันว่า “ดาวฤกษ์ที่ล้มเหลว (failed stars)” และด้วยเหตุนี้เองทำให้พวกมันมีค่ากำลังส่องสว่าง (luminosity) และอุณหภมูิที่ต่ำ อีกทั้งยังมีขนาดเล็ก ทำให้ถูกค้นพบในย่านคลื่นอินฟราเรดโดยทั่วไป

การจำแนกดาวแคระน้ำตาล

การวิเคราะห์เส้นสเปกตรัม (Spectrograph)

ปกตินักดาราศาสตร์จะใช้การวิเคราะห์เส้นสเปกตรัม (Spectrograph) ในการจัดหมวดหมู่ของดาว ตั้งแต่ดาวฤกษ์ไปจนถึงดาวแคระตามประเภทต่าง ๆ การวิเคราะห์เส้นสเปกตรัมสามารถทำได้โดยใช้เทคนิค Spectroscopy ซึ่งเป็นการแยกแสงที่ได้มาจากการสังเกตการณ์ด้วยการใช้เกรตติ้งเลี้ยวเบน (Diffraction grating) เพื่อให้เห็นค่าความเข้มแสงของแต่ละย่านคลื่น

รูปร่างของเส้นสเปกตรัมนั้นจะเปลี่ยนไปตามคุณสมบัติของดาว เช่น อุณหภูมิพื้นผิวของดาวและองค์ประกอบธาตุในดาว ค่าอุณหภูมิพื้นผิวของดาวจะขึ้นอยู่กับยอดคลื่นของเส้นสเปกตรัมที่มีรูปร่างตามกฎการแผ่รังสีของวัตถุดำ (Blackbody radiation) ส่วนค่าองค์ประกอบธาตุของดาวจะมีผลต่อการปรากฏของเส้นการดูดกลืน (Absorbtion line) ที่ความยาวคลื่นต่าง ๆบนเส้นสเปกตรัม ทำให้เราสามารถใช้การวิเคราะห์เส้นสเปกตรัมในการวัดค่าอุณหภูมิพื้นผิวของและความเป็นโลหะของดวงดาวได้ ซึ่งเราใช้ข้อมูลเหล่านี้ในการจำแนกชนิดดาวออกตาม spectrum class ในแต่ละหมวดหมู่ตามคุณสมบัติของดาว

Spectrum class มาตราฐานของดาวมีอยู่ 7 ประเภทหลัก ๆ ได้แก่ O, B, A, F, G, K และ M เรียงตามอุณหภูมิพื้นผิวจากร้อนสุดไปหาเย็นที่สุดดัง Figure 1. แต่เมื่อพิจารณาการมีอยู่ของดาวแคระน้ำตาลซึ่งมีอุณหภูมิต่ำกว่าประเภท M และมีค่าความเป็นโลหะที่มากกว่าดาวฤกษ์ทั่วไป จึงมีการแบ่งประเภทของ Spectrum class ย่อยเพิ่มขึ้นมาอีก 3 ประเภทคือ L, T และ Y ดังที่เห็นใน Table 1. ซึ่งในบางครั้งนักดาราศาสตร์ยังจัดว่าดาวแคระแดง (Red dwarf) ที่อยู่ในประเภท M ช่วงปลาย (M6-M9) เป็นดาวแคระแคระน้ำตาลด้วย ขึ้นอยู่กับ Absorbtion line ของดาวแคระแดง ว่ามีเส้นสเปกตรัมของธาตุ Li ที่เกิดมาจากปฎิกริยาฟิวชันของดิวเทอเรียมหรือไม่

การจำแนกดาวแคระแคระน้ำตาลด้วยวิธีการทาง Photometric

นอกจากการใช้เทคนิค Spectroscopy เพื่อจำแนกประเภทดาวแคระน้ำตาลแล้ว เราก็ยังสามารถใช้วิธีทาง Photometric ในการจำแนกประเภทดาวแคระน้ำตาลได้ด้วย โดยเทคนิคนี้ถูกเสนอขึ้นโดย Nathalie Skrzypek ซึ่งเริ่มต้นในปี 2013 และได้ตีพิมพ์ในปี 2015 (Skrzypek et al. 2015) ซึ่งจำแนกดาวแคระน้ำตาลโดยการ fit graph ตาม template ของ color index ดังที่แสดงใน Figure 2.

การหาค่า color index ทำได้โดยการคำนวณความแตกต่างของ photon flux ระหว่าง 2 ย่านคลื่นที่เราสนใจ โดยค่า photon flux สามารถหาได้จากการใช้ filter กรองแสงในคลื่นย่านอื่นออกไป ซึ่งย่านคลื่นที่ครอบคลุมในแต่ละ filter เป็นดังตัวอย่างใน Figure 3. จะเห็นได้ว่าเมื่อใช้ ‘V filter (เส้นสีเขียว)’ ค่า photon flux ที่วัดได้จะอยู่ในช่วงความยาวคลื่น 450 nm to 650 nm ทำให้เราจะได้ค่า Magnitude ของดาวดวงนั้นในย่านคลื่นนี้ การคำนวณค่า color index ทำได้โดยการนำ Magnitude ของ 2 filter มาลบกัน ในตัวอย่าง Figure 2. แกน y คือค่า color index ระหว่าง ‘i filter’ และ ‘Z filter’ เมื่อได้ค่า color index แล้ว Skrzypek et al. 2015 ได้ใช้วิธีการทางสถิติในการสร้าง template สำหรับ fit graph ระหว่างชนิดของดาวแคระน้ำตาลกับค่า color index โดยรายละเอียดสามารถอ่านเพิ่มเติมได้ในเปเปอร์ Skrzypek et al. 2015

ข้อมูลการสังเกตุการณ์ดาวแคระน้ำตาล

ในปี 2018 บทความวิจัย (Best et al. 2018) ได้รวบรวมข้อมูลดาวแคระในประเภท M, L, และ T จำนวน ~9000 ดวง จากโครงการสังเกตุการณ์ต่าง ๆ ได้แก่ PanSTARRS1 3π Survey (PS1), Two Micron All Sky Survey (2MASS), All-Wide-field Infrared Survey Explorer (AllWISE), และ Gaia DR1 โดยประกอบไปด้วยข้อมูลของดาวแต่ละดวง เช่น ประเภทของสเปกตรัม ,โฟโตเมตริก, Proper Motion และปริมาณทางฟิสิกส์อื่น ๆ บทความวิจัย Avdeeva 2023 ต้องการจำแนกดาวแคระน้ำตาลกับดาวฤกษ์ด้วยการใช้ Machine learning ซึ่งใช้ข้อมูลดาวแคระน้ำตาลจาก (Best et al. 2018) โดยจากนี้เราจะเรียกดาวแคระน้ำตาลว่าเป็น positive class หรือก็คือวัตถุที่เราสนใจ ส่วนข้อมูลของวัตถุอื่นที่ไม่ใช่ดาวแคระน้ำตาล หรือก็คือ negative class นั้นได้มาจากข้อมูลของโครงการสำรวจ Gaia DR3 ซึ่งประกอบด้วย ฐานข้อมูล Simbad ซึ่งมีดาวฤกษ์ใน O-M class รวมถึงฐานข้อมูลย่อยอื่น ๆตามที่บรรยายไว้ใน Avdeeva 2023 ในท้ายที่สุดจะมีข้อมูล positive class (ดาวแคระน้ำตาล ใน L-T class) จำนวน ~1600 ข้อมูล และ negative class (ดาวฤกษ์ใน O-M class) จำนวน ~4100 ข้อมูล ตามลิ้ง github ซึ่งเป็นข้อมูลที่เราจะใช้ในแบบฝึกหัดนี้

การเติมข้อมูล (Augmentation)

จากข้อมูลที่เรามีจะเห็นได้ว่าข้อมูลของดาวฤกษ์และวัตถุอื่นที่เราไม่สนใจ (ซึ่งเรา label เป็น negative class) นั้นมีจำนวนที่มากกว่าดาวแคระน้ำตาล (label เป็น positive class) อยู่ 2.5 เท่า ดังนั้นเราจึงจำเป็นต้องทำให้จำนวนข้อมูลดาวแคระน้ำตาลและดาวฤกษ์ในกลุ่มตัวอย่างของเรามีความสมดุลกัน ด้วยการทำให้ข้อมูลทั้งสองชุดนี้มีจำนวนที่ใกล้เคียงกัน โดยเราจะใช้การเพิ่มสัญญาณรบกวนในข้อมูล (noise) ตามการกระจายตัวแบบ Gaussian-distributed noise เพื่อปรับจำนวนข้อมูลของทั้งดาวแคระน้ำตาลและดาวฤกษ์ให้มีจำนวนที่ใกล้เคียงกัน

Gaussian-distributed noise

Gaussian-distributed noise จะถูกเขียนในรูป probability density function ที่มีการกระจายตัวแบบ Gaussian (หรือ normal distribution) ตามสมการ

$$ f(x) = \frac{1}{\sigma \sqrt{2\pi}} e^{-\frac{1}{2} \left(\frac{x-\mu}{\sigma}\right)^2} $$

โดย \(x\) คือ ขนาดของสัญญาณรบกวน, \(\sigma\) คือส่วนเบี่ยงเบนมาตรฐาน (standard deviation) ซึ่งจะกำหนดความกว้างของ probability density function และ \(\mu\) คือค่าเฉลี่ยของข้อมูล (mean) ซึ่งจะเป็นตัวกำหนดจุดกึ่งกลางของ probability density function ดังที่แสดงใน Figure 6.

การสร้าง Gaussian-distributed noise นั้นสามารถใช้คำสั่ง [random.normal (loc=0.0, scale=1.0, size=None)] numpy random.normal ใน library numpy โดยคำสั่งนั้นจะรับค่า parameter 3 ตัว คือ loc , scale และ size โดยที่ loc คือ ค่าเฉลี่ย (mean) หรือ ก็คือจุดกึ่งกลางของการกระจายตัวของ noise ตาม Gaussian distribution scale คือ ส่วนเบี่ยงเบนมาตรฐาน (\(\sigma\)) หรือก็คือความกว้างของการกระจายตัวของ noise size คือ ขนาดและรูปแบบของ array ที่ output ออกมา เมื่อเราได้ noise distribution มาแล้วก็ทำการใส่ 'noise' ลงไปในข้อมูลจริงๆ ซึ่งเราอาจจะเขียน function ในภาษา python ได้ดังนี้ 

			
import numpy as np

def add_gaussian_noise(data, mean, std): 
	"""
	ฟังก์ชันในการใส่ Gaussian noise ลงไปในชุดข้อมูล
  
	parameters:
	- data: ชุดข้อมูลที่ต้องการใส่ noise
	- mean: ค่าเฉลี่ย (mean) จุดกึ่งกลางของการกระจายตัวของ noise
	- std: ส่วนเบี่ยงเบนมาตรฐาน (σ) ความกว้างของการกระจายตัวของ noise

	output: ชูดข้อมูลที่ถูกใส่ noise แล้ว ซึ่งจะเป็น array รูปแบบเดียวกับชุดข้อมูลที่ถูกใส่เข้ามา

	"""
	
	# Generating Gaussian noise with the same shape as the input data
	noise = np.random.normal(loc = mean, scale = std, size = data.shape)
	
	"""
	การสร้างฟังชันก์การกระจายตัวของ noise โดยให้ 
	loc = mean
	scale = std
	size = รูปแบบของ output array ให้เป็นแบบเดียวกับชุดข้อมูลที่ถูกใส่เข้ามา
	"""

	# Adding the Gaussian noise to the input data
	noisy_data = data + noise
	
	"""
	เมื่อได้ noise distribution แล้ว ก็นำ noise ใส่ในชุดข้อมูลของเรา
	สุดท้ายเราก็จะได้ ข้อมูลที่ถูกใส่ noise คือ "noisy_data"
	"""
	
	return noisy_data

ในแบบฝึกหัดนั้น เราจะใส่ Gaussian-distributed noise ในแต่ละ feature โดยใช้ค่าเฉลี่ย (mean) ของข้อมูลใน feature เป็นค่า \(\mu\) ใน Gaussian distribution และส่วนเบี่ยงเบนมาตราฐานของค่า error ของแต่ feature เป็นค่า \(\sigma\) ใน Gaussian distribution ตัวอย่างโด้ดที่แสดงรายละเอียดขั้นตอนการเติมข้อมูล (Augmentation) นั้นสามารถดูได้ที่นี่: Colab

การใช้ Machine learning ในการจำแนกดาวแคระน้ำตาล

ต้นไม้การตัดสินใจ (Decision tree)

Decision tree เป็นโมเดลที่สามารถใช้ได้ในการแก้ปัญหาการจำแนกประเภท (classification) และการถดถอย (regression) โดยอิงตามลำดับขั้นตอนของต้นไม้ที่สร้างขึ้นจากข้อมูลที่มีอยู่โดยใช้ฟังชันก์ทางคณิตศาสตร์ Decision tree เป็นอัลกอริทึมที่ใช้ในการทำนายผลลัพธ์ของการตัดสินใจ (ในกรณีของเราคือ การจำแนกประเภทของดาวแคระน้ำตาล) โดยโครงสร้างของ Decision tree จะประกอบไปด้วยโหนด (node) ต่าง ๆเชื่อมหากัน

เราเรียกจุดเริ่มต้นของ Decision tree ว่า "ราก" (root) ซึ่งเป็นเงื่อนไขเริ่มต้นในการแบ่งข้อมูลออกเป็นกลุ่มย่อย โหนดย่อยที่เชื่อมต่อกับราก เรียกว่า "โหนดลูก" (child nodes) ซึ่งเป็นเงื่อนไขการแบ่งข้อมูลเพิ่มเติม โหนดที่อยู่ ณ จุดสิ้นสุดของแต่ละเงื่อนไขคือ "โหนดปลาย" (leaf node) ซึ่งจะให้ผลลัพธ์ของการทำนายออกมา

ตัวอย่างใน Figure 7. แสดง decision tree ในการจำแนกสัตว์ 3 ตัว ประกอบด้วย dog, spider, และ penguin โดยโหนดบนสุดของ decision tree คือราก และมีโหนดของเงื่อนไขเป็นโหนดลูก โหนดตรงปลายคือผลการทำนาย ซึ่งในตัวอย่างนี้คือประเภทของสัตว์

Random forest regression

Random Forest Regression เป็น Machine learning ที่ใช้กลุ่มก้อน (ensemble) ซึ่งประกอบไปด้วย Decision tree หลายต้นในการทำนายข้อมูล โดยการสร้างกลุ่มก้อนของ Decision tree นั้นมาจากกระบวนการ Bootstrapping และ Attribute sampling ซึ่งสองกระบวนการนี้เป็นส่วนสำคัญที่ทำให้ได้ Decision tree แต่ละต้นที่มีความแตกต่างกัน

Bootstrapping คือการสุ่มสับเซตข้อมูลจากชุดข้อมูลต้นฉบับ โดยใช้การสุ่มแบบใส่คืน (Sampling with Replacement) ซึ่งมีความหมายทางคณิตศาสตร์ว่า การสุ่มแต่ละครั้งเป็นอิสระต่อกัน (independent) หมายความว่าข้อมูลบางตัวอาจถูกสุ่มมาหลายครั้ง แต่บางตัวก็อาจจะไม่ถูกสุ่มมาเลย โดยจำนวนครั้งที่ทำการสุ่มนั้นมีค่าเท่ากับขนาดของข้อมูลต้นฉบับ หมายความว่าจำนวนของข้อมูลที่ได้จากการทำ Bootstrapping จะมีขนาดเท่ากันกับจำนวนข้อมูลต้นฉบับ

ตัวอย่างใน Table 2. แสดงจำนวนข้อมูลที่ได้จากการทำ Bootstrapping จะเห็นได้ว่าในบางครั้งเราได้ข้อมูลบางตัวที่ปรากฏซ้ำ (Bootstrap round 1 ได้ sample 4 ออกมา 2 ตัว) ในขณะที่บางครั้งมีข้อมูลบางตัวที่หายไป (Bootstrap round 2 ได้ sample 2, 4, และ 6 ออกมา 0 ตัว)

ผลของจำนวนของข้อมูลจากการทำ Bootstrapping จำนวน 3 ครั้ง ของข้อมูล 6 ตัว

Attribute sampling คือการสุ่มเลือก feature ที่มีความเหมาะในขั้นตอนการสร้างโหนดลูกที่ระบุเงื่อนไขการแบ่งข้อมูล หมายความว่าในขั้นตอนการสร้างโหนดลูกไม่ได้พิจารณาทุก feature แต่จะสุ่มเลือกแค่บาง feature มาพิจารณาเท่านั้น ก่อนทำการหา feature ที่ดีที่สุด

ตัวอย่างใน Figure 8. แสดง Attribute sampling ของข้อมูลที่มี 5 feature: f1, f2, f3, f4, และ f5 โดย feature ที่สุ่มมาใช้ในการพิจารณาคือสีน้ำเงิน ส่วนสีขาวคือ feature ที่ไม่ได้พิจารณา กรอบสีแดงแสดง feature ที่ดีที่สุดที่ถูกเลือกในการสร้างเงื่อนไข

สับเซตข้อมูลที่ได้จากสองกระบวนการนี้จะถูกนำมาสร้าง Decision trees จากนั้นเราทำกระบวนการเหล่านี้ซ้ำไปเรื่อย ๆ เพื่อสร้าง Decision trees ขึ้นมาหลายต้นเพื่อรวมเป็นกลุ่มก้อน (ensemble) เข้าด้วยกันเป็น Random Forest Model (เรียกกระบวนการนี้ถูกเรียกว่า Ensemble learning) ซึ่งมีความแม่นยำมากกว่า และเสี่ยงต่อการเกิด Overfitting น้อยกว่า เมื่อเปรียบเทียบกับการใช้ Decision tree เพียงต้นเดียวดังที่แสดงใน Figure 9.
