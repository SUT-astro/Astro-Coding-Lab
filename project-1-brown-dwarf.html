ดาวแคระน้ำตาล (Brown Dwarfs)

ดาวแคระน้ำตาลคืออะไร

ดาวแคระน้ำตาลคือดาวที่มีมวลมากกว่าดาวเคราะห์ แต่น้อยกว่าดาวฤกษ์ ในทางดาราศาสตร์ ดาวแคระน้ำตาลถูกจัดว่าเป็นวัตถุมวลต่ำกว่าดาวฤกษ์ (substellar object) ซึ่งถูกคาดการณ์การมีอยู่ทางทฤษฎีครั้งแรกโดย Dr. Shiv S. Kumar ในช่วงปี 1960s ในช่วงแรก เราเรียกดาวแคระห์น้ำตาลว่า “ดาวแคระดำ” อันเนื่องมาจากขนาดที่เล็กและอุณหภูมิที่ต่ำ ทำให้ยากต่อการค้นพบจากการสังเกตการณ์ จนกระทั้งในปี 1994-1995 มีการค้นพบดาวแคระน้ำตาล 2 ดวงคือ Gliese 229b and Teide 1 จากการสังเกตการณ์ในย่านคลื่นอินฟราเรด

ทำไมดาวแคระน้ำตาลจึงถูกค้นพบได้ยาก?

เนื่องจากมวลของดาวแคระน้ำตาลนั้นมีค่ามากกว่ามวลของดาวเคราะห์ แต่น้อยกว่าดาวฤกษ์ (มีมวลระหว่าง 13 ถึง 80 เท่าของดาวพฤหัสบดี) ซึ่งอยู่ในช่วงที่ทำให้เกิดปฎิกริยาฟิวชันของดิวเทอเรียม (Deuterium fusion) ที่แกนกลางของดาว แต่ไม่มากพอที่จะทำให้เกิดเกิดปฎิกริยาฟิวชันของไฮโดรเจนแบบดาวฤกษ์ (มีมวล > 80 เท่าของดาวพฤหัส) ทำให้บางครั้งนักดาราศาสร์เรียกมันว่า “ดาวฤกษ์ที่ล้มเหลว (failed stars)” และด้วยเหตุนี้เองทำให้พวกมันมีค่ากำลังส่องสว่าง (luminosity) และอุณหภมูิที่ต่ำ อีกทั้งยังมีขนาดเล็ก ทำให้ถูกค้นพบในย่านคลื่นอินฟราเรดโดยทั่วไป

การจำแนกดาวแคระน้ำตาล

ปกตินักดาราศาสตร์จะใช้การวิเคราะห์เส้นสเปกตรัม (Spectrograph) ในการจัดหมวดหมู่ของดาว ตั้งแต่ดาวฤกษ์ไปจนถึงดาวแคระตามประเภทต่าง ๆ การวิเคราะห์เส้นสเปกตรัมสามารถทำได้โดยใช้เทคนิค Spectroscopy ซึ่งเป็นการแยกแสงที่ได้มาจากการสังเกตการณ์ด้วยการใช้เกรตติ้งเลี้ยวเบน (Diffraction grating) เพื่อให้เห็นค่าความเข้มแสงของแต่ละย่านคลื่น

รูปร่างของเส้นสเปกตรัมนั้นจะเปลี่ยนไปตามคุณสมบัติของดาว เช่น อุณหภูมิพื้นผิวของดาวและองค์ประกอบธาตุในดาว ค่าอุณหภูมิพื้นผิวของดาวจะขึ้นอยู่กับยอดคลื่นของเส้นสเปกตรัมที่มีรูปร่างตามกฎการแผ่รังสีของวัตถุดำ (Blackbody radiation) ส่วนค่าองค์ประกอบธาตุของดาวจะมีผลต่อการปรากฏของเส้นการดูดกลืน (Absorbtion line) ที่ความยาวคลื่นต่าง ๆบนเส้นสเปกตรัม ทำให้เราสามารถใช้การวิเคราะห์เส้นสเปกตรัมในการวัดค่าอุณหภูมิพื้นผิวของและความเป็นโลหะของดวงดาวได้ ซึ่งเราใช้ข้อมูลเหล่านี้ในการจำแนกชนิดดาวออกตาม spectrum class ในแต่ละหมวดหมู่ตามคุณสมบัติของดาว

Spectrum class มาตราฐานของดาวมีอยู่ 7 ประเภทหลัก ๆ ได้แก่ O, B, A, F, G, K และ M เรียงตามอุณหภูมิพื้นผิวจากร้อนสุดไปหาเย็นที่สุดดังรูปที่ 1. แต่เมื่อพิจารณาการมีอยู่ของดาวแคระน้ำตาลซึ่งมีอุณหภูมิต่ำกว่าประเภท M และมีค่าความเป็นโลหะที่มากกว่าดาวฤกษ์ทั่วไป จึงมีการแบ่งประเภทของ Spectrum class ย่อยเพิ่มขึ้นมาอีก 3 ประเภทคือ L, T และ Y ดังที่เห็นในรูปที่ 2. ซึ่งในบางครั้งนักดาราศาสตร์ยังจัดว่าดาวแคระแดง (Red dwarf) ที่อยู่ในประเภท M ช่วงปลาย (M6-M9) เป็นดาวแคระแคระน้ำตาลด้วย ขึ้นอยู่กับ Absorbtion line ของดาวแคระแดง ว่ามีเส้นสเปกตรัมของธาตุ Li ที่เกิดมาจากปฎิกริยาฟิวชันของดิวเทอเรียมหรือไม่

การจำแนกดาวแคระแคระน้ำตาลด้วยวิธีการทาง Photometric

นอกจากการใช้เทคนิค Spectroscopy เพื่อจำแนกประเภทดาวแคระน้ำตาลแล้ว เราก็ยังสามารถใช้วิธีทาง Photometric ในการจำแนกประเภทดาวแคระน้ำตาลได้ด้วย โดยเทคนิคนี้ถูกเสนอขึ้นโดย Nathalie Skrzypek ซึ่งเริ่มต้นในปี 2013 และได้ตีพิมพ์ในปี 2015 (Skrzypek et al. 2015) ซึ่งจำแนกดาวแคระน้ำตาลโดยการ fit graph ตาม template ของ color index ดังที่แสดงในรูปที่ 3.

การหาค่า color index ทำได้โดยการคำนวณความแตกต่างของ photon flux ระหว่าง 2 ย่านคลื่นที่เราสนใจ โดยค่า photon flux สามารถหาได้จากการใช้ filter กรองแสงในคลื่นย่านอื่นออกไป ซึ่งย่านคลื่นที่ครอบคลุมในแต่ละ filter เป็นดังตัวอย่างในรูปที่ 4. จะเห็นได้ว่าเมื่อใช้ ‘V filter (เส้นสีเขียว)’ ค่า photon flux ที่วัดได้จะอยู่ในช่วงความยาวคลื่น 450 nm to 650 nm ทำให้เราจะได้ค่า Magnitude ของดาวดวงนั้นในย่านคลื่นนี้ การคำนวณค่า color index ทำได้โดยการนำ Magnitude ของ 2 filter มาลบกัน ในตัวอย่างรูปที่ 3. แกน y คือค่า color index ระหว่าง ‘i filter’ และ ‘Z filter’ เมื่อได้ค่า color index แล้ว Skrzypek et al. 2015 ได้ใช้วิธีการทางสถิติในการสร้าง template สำหรับ fit graph ระหว่างชนิดของดาวแคระน้ำตาลกับค่า color index โดยรายละเอียดสามารถอ่านเพิ่มเติมได้ในเปเปอร์ Skrzypek et al. 2015

ข้อมูลการสังเกตุการณ์ดาวแคระน้ำตาล

ในปี 2018 บทความวิจัย (Best et al. 2018) ได้รวบรวมข้อมูลดาวแคระในประเภท M, L, และ T จำนวน ~9000 ดวง จากโครงการสังเกตุการณ์ต่าง ๆ ได้แก่ PanSTARRS1 3π Survey (PS1), Two Micron All Sky Survey (2MASS), All-Wide-field Infrared Survey Explorer (AllWISE), และ Gaia DR1 โดยประกอบไปด้วยข้อมูลของดาวแต่ละดวง เช่น ประเภทของสเปกตรัม ,โฟโตเมตริก, Proper Motion และปริมาณทางฟิสิกส์อื่น ๆ บทความวิจัย Avdeeva 2023 ต้องการจำแนกดาวแคระน้ำตาลกับดาวฤกษ์ด้วยการใช้ Machine learning ซึ่งใช้ข้อมูลดาวแคระน้ำตาลจาก (Best et al. 2018) โดยจากนี้เราจะเรียกดาวแคระน้ำตาลว่าเป็น positive class หรือก็คือวัตถุที่เราสนใจ ส่วนข้อมูลของวัตถุอื่นที่ไม่ใช่ดาวแคระน้ำตาล หรือก็คือ negative class นั้นได้มาจากข้อมูลของโครงการสำรวจ Gaia DR3 ซึ่งประกอบด้วย ฐานข้อมูล Simbad ซึ่งมีดาวฤกษ์ใน O-M class รวมถึงฐานข้อมูลย่อยอื่น ๆตามที่บรรยายไว้ใน Avdeeva 2023 ในท้ายที่สุดจะมีข้อมูล positive class (ดาวแคระน้ำตาล ใน L-T class) จำนวน ~1600 ข้อมูล และ negative class (ดาวฤกษ์ใน O-M class) จำนวน ~4100 ข้อมูล ตามลิ้ง https://github.com/iamaleksandra/ML-Brown-Dwarfs ซึ่งเป็นข้อมูลที่เราจะใช้ในแบบฝึกหัดนี้

การเติม(Augmentation) ข้อมูลด้วย Gaussian-distributed noise

จากข้อมูลที่เรามีจะเห็นได้ว่าข้อมูลของดาวฤกษ์และวัตถุอื่นที่เราไม่สนใจ (ซึ่งเรา label เป็น negative class) นั้นมีจำนวนที่มากกว่าดาวแคระน้ำตาล (label เป็น positive class) อยู่ 2.5 เท่า ดังนั้นเราจึงจำเป็นต้องทำให้จำนวนข้อมูลดาวแคระน้ำตาลและดาวฤกษ์ในกลุ่มตัวอย่างของเรามีความสมดุลกันด้วยการทำให้ข้อมูลทั้งสองชุดนี้มีจำนวนที่ใกล้เคียงกัน โดยเราจะใช้การเพิ่มสัญญาณรบกวนในข้อมูล (noise) ตามการกระจายตัวแบบ Gaussian-distributed noise เพื่อปรับจำนวนข้อมูลของทั้งดาวแคระน้ำตาลและดาวฤกษ์ให้มีจำนวนที่ใกล้เคียงกัน Gaussian-distributed noise

Gaussian-distributed noise จะถูกเขียนในรูป probability density function ที่มีการกระจายตัวแบบ Gaussian (หรือ normal) ตามสมการ

โดย x คือ ขนาดของสัญญาณรบกวน, σ คือส่วนเบี่ยงเบนมาตรฐาน (standard deviation) ซึ่งจะกำหนดความกว้างของ probability density function และ μ คือค่าเฉลี่ยของข้อมูล (mean) ซึ่งจะเป็นตัวกำหนดจุดกึ่งกลางของ probability density function ดังที่แสดงในรูปที่ 6.

การสร้าง Gaussian-distributed noise นั้นสามารถใช้คำสั่ง [ random.normal (loc=0.0, scale=1.0, size=None) ] https://numpy.org/doc/stable/reference/random/generated/numpy.random.normal.html ใน library numpy โดยคำสั่งนั้นจะรับค่า parameter 3 ตัว คือ loc , scale และ size โดยที่ loc คือ ค่าเฉลี่ย (mean) หรือ ก็คือจุดกึ่งกลางของการกระจายตัวของ noise ตาม Gaussian distribution scale คือ ส่วนเบี่ยงเบนมาตรฐาน (σ) หรือก็คือความกว้างของการกระจายตัวของ noise size คือ ขนาดและรูปแบบของ array ที่ output ออกมา เมื่อเราได้ noise distribution มาแล้วก็ทำการใส่ 'noise' ลงไปในข้อมูลจริงๆ ซึ่งเราอาจจะเขียน function ในภาษา python ได้ดังนี้ 

        
          import numpy as np

          def add_gaussian_noise(data, mean, std): 
              """
              ฟังก์ชันในการใส่ Gaussian noise ลงไปในชุดข้อมูลใดๆ
          
              พารามิเตอร์:
              - data: คือชุดข้อมูลที่ต้องการใส่ noise
              - mean: ค่าเฉลี่ย(mean) หรือ ก็คือจุดกึ่งกลางของ การกระจายตัวของnoise (noise distribution)
              - std: ส่วนเบี่ยงเบนมาตรฐาน(σ) หรือก็คือความกว้างของการกระจายตัวของnoise
          
              output : ชูดข้อมูลที่ถูกใส noise แล้ว ซึ่งจะเป็น array รูปแบบเดียวกับชุดข้อมูลที่ถูกใส่เข้ามา 
          
              """
              # Generating Gaussian noise with the same shape as the input data
              noise = np.random.normal(mean, std, size=data.shape)
              """
              การสร้างฟังชันก์การกระจายตัวของnoise โดยให้ 
              loc = mean ที่ถูกใส่เข้ามา, 
              scale = std ที่่ถูกใส่เข้ามา
              size = รูปแบบ array แบบเดียวกับชุดข้อมูลที่ถูกใส่เข้ามา
              
              """
          
              # Adding the Gaussian noise to the input data
              noisy_data = data + noise
              """
              เมื่อได้ noise distribution แล้ว ก็นำnoise ใส่ในชุดข้อมูลของเรา
              สุดท้ายเราก็จะได้ ข้อมูลที่ถูกใส่noise คือ " noisy_data " ที่เป็น function ของ data+noise
              """
              return noisy_data

โดยในส่วนของแบบฝึกหัดนั้น เราจะใส่ noise ใน feature โดยใช่ค่า ค่าเฉลี่ย(mean)และส่วนเบี่ยงเบนมาตราฐานของแต่ละ error ของแต่ fearture นั้นๆ ซึ่งตัวอย่างโด้ดถูกจะแสดงใน Link

การจัดการกับข้อมูลที่หายไป(Missing value)

Decision tree(ต้นไม้การตัดสิดใจ)

Decision tree learning เป็น machine model ที่ใช้ในการแก้ปัญหา classification หรือ resgression โดยอิงตามลำดับขั้นตอนของต้นไม้ที่สร้างขึ้นจากข้อมูลที่มีอยู่โดยใช้ฟังชันก์ทางคณิตศาสตร์ Decision tree เป็นอัลกอริทึมที่ใช้ในการทำนายผลลัพธ์ของการตัดสินใจ(ในกรณีของเราคือการจำแนกดาวแคระน้ำตาล) โดยมีโครงสร้างของต้นไม้การตัดสินใจที่ประกอบด้วยโหนด (nodes) และเส้นเชื่อม (branches) ซึ่งแทนคำตอบที่เป็นไปได้ทั้งหมด เราสามารถเริ่มต้นด้วยโหนดหลักที่เรียกว่า "โหนดราก" (root node) ที่จะแบ่งข้อมูลออกเป็นกลุ่มย่อยๆ ตามเงื่อนไขที่กำหนด หลังจากนั้นเราจะมีโหนดย่อยที่เชื่อมต่อกับโหนดราก ซึ่งเรียกว่า "โหนดลูก" (child nodes) ที่จะทำการตัดสินใจเพิ่มเติม ตามลำดับเรื่อยๆ จนกระทั่งถึงโหนดที่เรียกว่า "โหนดปลาย" (leaf node) ที่จะให้คำตอบสุดท้าย ยกตัวอย่างเช่นดังรูปที่ 8 แสดงถึง decision tree ของการไปเที่ยวของนาย A โดย โหนดสีน้ำเงินแรกสุด(บนสุด) ของdecision tree คือ โหนดราก และโหนดสีเงินอื่นๆคือโหนดลูก โดยโหนดสีเขียวคือโหนดปลาย หรือก็คือผลลัพย์ของการตัดใจ

Random forest regression

Random Forest Regression เป็นmachine learning ซึ่งมีการรวม Decision trees หลายๆตัวเพื่อทำนายผลลัพธ์ของข้อมูล โดยการสร้าง Decision tree ที่ไม่เหมือนกัน แต่มีลำดับขั้นตอนที่คล้ายกัน ซึ่งมีกระบวนการ Bootstrapping เป็นส่วนสำคัญที่นำใน Random Forest Regression เพื่อการสร้างความแตกต่างในแต่ล่ะ Decision tree โดยในขั้นตอนนี้จะทำการสุ่มสับเซตข้อมูลจากชุดข้อมูลต้นฉบับ โดยการสุ่มด้วยการคืน (with replacement) ซึ่งหมายความว่าข้อมูลบางส่วนอาจถูกสุ่มมาหลายครั้ง และบางส่วนก็อาจจะไม่ถูกนำมาเลย โดยจะมีการสุ่มตัวอย่างข้อมูลจากชุดข้อมูลต้นฉบับ และนำมาสร้างDecision trees จากนั้นเราทำกระบวนการนี้ซ้ำไปเรื่อยๆ เพื่อสร้าง Decision trees ที่หลากหลายขึ้น จากนั้นทำการรวมทุกๆ Decision trees เข้าด้วยกันเป็น Random Forest Model(เรียกกระบวนการนี้ถูกเรียกว่า Ensemble learning) ที่มีความแม่นยำและทนทานต่อการเกิด Overfitting(การที่เหมาะสมกับชุดข้อมูลที่ถูกนำไปฝึกฝนเท่านั้น เมื่อนำไปทำนายกับชุดอื่นๆ อาจจะให้ผลลัพพ์ที่แย่มากๆได้) เเละสามารถทำนายผลลัพธ์ในข้อมูลที่ไม่เคยเห็นมาก่อนได้ดี ดังที่แสดงในรูปที่ X
